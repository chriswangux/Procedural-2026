Case Study

Building Procedural Design

How AI agent teams turned a presentation deck into a 21-section interactive web experience

Built with Claude Code View the live experience →

It started with a Google Slides presentation. A designer exploring the intersection of procedural systems and creative practice had built a deck around one provocation:

"Designing the machine that designs the design."

The deck was dense with ideas — particle simulations, generative architecture, semantic lighting, style transfer, emergent behavior. Each slide a window into how procedural thinking could reshape creative work. But slides are static. The ideas in this deck begged to be experienced.

The question became: could we turn this deck into something interactive, where every idea is not just explained but demonstrated?

Phase 1

The First Prototype

The first pass took an aggressive approach: spawn five AI agents in parallel, each responsible for building one interactive section from scratch. No shared code. No design system. Just a prompt and a PDF.

Each agent independently interpreted a concept from the deck and built a fully interactive demo:

Particle Systems Planet Generator Spider Web Sim Semantic Manipulation Generative Design Grids

Five agents working simultaneously, each producing a self-contained interactive section with its own canvas, controls, and animation loop. The results were stitched together into a single scrollable page with a cohesive dark theme and smooth navigation.

5 × 5 Five agents, five sections, roughly 5,500 lines of code — built simultaneously in a single parallel run.

It worked. But it only scratched the surface.

Phase 2

Going Exhaustive

The first prototype captured about five of the deck's twenty-five-plus ideas. The user's direction was clear: be exhaustive. Don't skip a single concept.

A systematic audit of the original presentation identified twenty missing ideas that hadn't been built. These were grouped into coherent themes and scoped into twelve new interactive sections, each requiring its own simulation, rendering approach, or interactive model.

This wasn't incremental. It was a ground-up expansion — nearly tripling the scope while maintaining the visual language and interaction patterns established in Phase 1.

The Full Build

The Meta Connection

There's something recursive about this project. The deck itself argues for procedural, AI-augmented design — and that's exactly how it was built. The process mirrors the ideas:

  • Human in the Loop — A human directed the vision, set the quality bar, and decided what to build. The agents filled in the details, wrote the code, and handled the combinatorial complexity.
  • Exploration at Scale — Six agents exploring different design spaces simultaneously is exactly the "explore more, faster" thesis of the deck. No single person could have prototyped twelve interactive simulations in parallel.
  • Procedural at Every Level — Each interactive section generates its visuals from rules and parameters, not hand-crafted assets. The spiders build webs from tension rules. The cities grow from architectural grammars. The particles follow force fields.
  • Participatory, Not Passive — Viewers don't just read about procedural design — they drag sliders, click canvases, and watch systems respond. The ideas are experienced, not explained.

The medium became the message.
